通过真实代码理解二者如何影响**性能与安全性**。

---

## 示例 1：noexcept 影响 vector 扩容

```cpp
struct A {
    A() = default;
    A(A&&) noexcept = default; // 关键
};

vector<A> v;
v.push_back(A{});
```

说明：

* 扩容时使用 move，而非 copy

---

## 示例 2：没有 noexcept 的代价

```cpp
struct B {
    B(B&&) { }
};

vector<B> v;
v.push_back(B{}); // 可能触发拷贝
```

---

## 示例 3：explicit 防止隐式构造

```cpp
struct File {
    explicit File(const char* path);
};

File f("a.txt");   // ✅
// File f2 = "a.txt"; // ❌
```

---

## 示例 4：缺少 explicit 的隐患

```cpp
struct Num {
    Num(int x);
};

void foo(Num n);

foo(10); // ❌ 隐式构造
```

---

## 示例 5：explicit + 工厂函数

```cpp
struct A {
    explicit A(int);
    static A create(int x) {
        return A(x);
    }
};
```

---

## 工程实践总结

* noexcept 是性能提示
* explicit 是安全阀
