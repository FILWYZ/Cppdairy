## 1. STL 算法库总览

C++ STL 算法位于：

```cpp
#include <algorithm>
```

核心特点：

* **算法与容器解耦**（Algorithm + Iterator）
* 基于 **迭代器区间 [first, last)**
* 通用、可复用、可组合

> STL 算法的设计思想：
> **对区间做操作，而不是对容器本身做操作**。

---

## 2. `sort` —— 排序算法

### 2.1 基本用法

```cpp
sort(begin, end);
sort(begin, end, comp);
```

* 默认：`operator<`
* 可传入自定义比较器

---

### 2.2 底层实现（重要）

`std::sort` 并不是单一算法，而是 **Introsort**：

* 快速排序（Quick Sort）
* 堆排序（Heap Sort）
* 插入排序（Insertion Sort，小规模）

目的：

* 平均性能快
* **最坏复杂度 O(n log n)**（避免快排退化）

---

### 2.3 复杂度与特性

| 特性    | 说明         |
| ----- | ---------- |
| 时间复杂度 | O(n log n) |
| 是否稳定  | ❌ 不稳定      |
| 随机访问  | ✅ 必须       |

> 因此 `sort` **不能用于 list**。

---

## 3. `find` —— 线性查找

### 3.1 基本用法

```cpp
auto it = find(begin, end, value);
```

* 返回第一个等于 `value` 的迭代器
* 未找到返回 `end`

---

### 3.2 复杂度分析

* 时间复杂度：`O(n)`
* 顺序遍历

⚠️ 注意：

* 对 `map / unordered_map` 使用的是 **成员函数 `find`**，不是算法版 `find`

---

## 4. `transform` —— 区间变换

### 4.1 单输入区间

```cpp
transform(in_begin, in_end, out_begin, op);
```

作用：

* 对每个元素执行 `op`
* 结果写入输出区间

---

### 4.2 双输入区间

```cpp
transform(b1, e1, b2, out, op);
```

* 常用于向量运算

---

### 4.3 复杂度

* 时间复杂度：`O(n)`
* 不改变输入区间（除非 in-place）

---

## 5. 算法 + 容器的配合原则

| 算法        | 迭代器要求                  | 典型容器          |
| --------- | ---------------------- | ------------- |
| sort      | RandomAccessIterator   | vector, deque |
| find      | InputIterator          | 所有容器          |
| transform | Input + OutputIterator | 所有顺序容器        |

---

## 6. 常见误区

1. 对 `list` 使用 `sort`（❌）
2. 用算法版 `find` 查 `map`（低效）
3. 忘记给 `transform` 预留输出空间

---

## 7. 面试一句话总结

> `sort` 基于 Introsort，要求随机访问；
> `find` 是线性查找；
> `transform` 是区间映射算法，体现 STL 算法与容器解耦的思想。
