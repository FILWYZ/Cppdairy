## 题目 1（必考）

❓ 为什么 C++ 不推荐 malloc？

### ✅ 面试级答案

1. malloc 不调用构造 / 析构
2. 破坏 RAII 资源管理
3. 类型不安全
4. 不支持多态
5. 不利于异常安全

---

## 题目 2

```cpp
Test* t = (Test*)malloc(sizeof(Test));
```

❓ 这是不是一个合法的 C++ 对象？

### ✅ 答案解析

* ❌ 不是
* 只是原始内存
* 构造函数未执行

## 题目 3（进阶）

❓ 那什么时候 C++ 会用 malloc？

### ✅ 答案要点

* 底层库
* 内存池实现
* placement new 场景

## Day 2 自检清单

* [ ] 我能说清 new 的三步
* [ ] 我知道 malloc 的边界
* [ ] 我理解 RAII 的意义


*我能说清 new 的三步
>new = 申请内存（operator new）→ 调用构造函数 → 返回类型化指针，失败抛异常（而不是返回 nullptr）。

*我知道 malloc 的边界
>malloc 只负责分配原始内存，不初始化、不调用构造/析构，返回 void*，必须手动管理类型和生命周期。

*我理解 RAII 的意义
>资源的获取即初始化，把资源（内存、文件、锁）的生命周期绑定到对象上，用构造函数获取、析构函数释放，避免泄漏和异常不安全。
